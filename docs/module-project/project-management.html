Project Management

Project Creation​

This section demonstrates how to establish a project hierarchical structure using the scenario of an apartment water leak detection project.

Apartment Water Leak Detection System
├── North Building A
│   └── F1-001
│   └── F1-002
│   └── F2-003

In this example:

Use Apartment Water Leak Detection System as a primary project. Use North Building A as a secondary project. Use F1-001, F1-002, F2-003 as tertiary projects (rooms).

Step1: Create a Primary Project "Apartment Water Leak Detection System"

Click Project Management to enter the project management page. Click the PROJECT button on the right. Fill in the project information in the pop-up form. Click the Save button to complete project creation.

info

The contact information in the project form can be used for action configuration as message recipients. Radius service reminders will be sent to the primary project contacts via SMS or email.

Extended Content:

Device Owner: The contact person directly associated with the device and the users directly related to the project. For example, if the F1-001 project has a water immersion device, and it detects a leak, and the message recipient is set as Device Owner in the AP, then the contact person listed in the F1-001 project, as well as users associated with F1-001, will receive notifications (project user management).

Project Owner: The contact persons and users of all higher-level projects directly associated with the device. For example, if the F1-001 project has a water immersion device, and it detects a leak, and the message recipient is set as Project Owner in the AP, then the contacts listed in the North Building A, Apartment Water Leak Detection System project, and associated users will receive notifications.

Step2: Create a Secondary Project North Building A

  1. Click Project Management to enter the project management page.
  2. Click on the primary project Apartment Water Leak Detection System created in Step1 in the project tree on the left.
  3. Click the SUB-PROJECT button on the right.
  4. Fill in the project information in the pop-up form.
  5. Click the Save button to complete project creation.

Step3: Repeat Step 2 to complete projects F1-001, F1-002, F2-003 creation

Project Editing​

  1. Click on Project Management to enter the project management page.
  2. Find the project to be modified from the right-hand list by clicking on the project count on the left.
  3. Click the Operation button of the project that needs modification, and from the dropdown list, click Edit.
  4. Change the project information in the pop-up form.
  5. Click the Update button to complete the project modification.

Project Deletion​

  1. Click on Project Management to enter the project management page.
  2. Find the project to be deleted from the right-hand list by clicking on the project count on the left.
  3. Click the Operation button of the project that needs to be deleted, and from the dropdown list, click Delete.
  4. Click the Delete button in the confirmation pop-up to delete the project.
warning

Here's the translated text into English, keeping the original formatting:

Note: The project deletion operation is irreversible. Once a project is deleted, the following data will also be deleted. Please proceed with caution.

* Sub-projects
* Project-associated equipment and sub-project associated equipment
* Project-associated groups and self-managed project groups
* Project-associated action configurations and sub-project associated action configurations
* Project-associated users, and sub-project associated users
* Project-associated alarms, and sub-project associated alarms

User and Permission Management​

Users and roles can be created at every level of the project.

  1. Click on Project Management to enter the project management page.
  2. Find the project to operate on from the right-hand list by clicking on the project count on the left.
  3. Click the Operation button of the project that needs modification, and from the dropdown list, click User Permissions.

User Creation​

  1. Click the Create Button.
  2. Fill in user information in the pop-up form.
  3. Click the Create Button to complete user creation.
info

If allowing user login is selected, the following details also need to be filled in:

  1. Account Name
  2. Login Password
  3. Account Role (please create the role first)

Note: Users of the current project cannot use roles created in higher-level projects.

User Invitation​

  1. Click the Invite Button.
  2. Fill in the invited user's email and role in the pop-up form.
  3. Click the Send button.
  4. The invited user receives the email and clicks on the invitation link to accept the invitation (create a new account or use an existing one).

User Editing​

  1. Click the Operation button of the user record that needs to be changed, and select Edit from the dropdown menu.
  2. Change user information in the pop-up form.
  3. Click Update to complete the user information change.
warning

User information added to the project by invitation cannot be modified.

User Deletion​

  1. Click the Operation button of the user record that needs to be deleted, and select Delete from the dropdown menu.
  2. Click Delete to complete user deletion.
warning

Users added to the project through invitation cannot be deleted, they can only be removed from the project.

Creation and Invitation​

  1. If users wish to manage multiple projects through one account, please add them to the projects using the invitation method.
  2. Administrators cannot change the information of users added to the project through invitation, they can only change the user's role in the project.
  3. Administrators cannot delete users added to the project through invitation, they can only remove the user from the project.
  4. Users added through the invitation method, by default, can log in to Raedius.

Role Creation​

Roles created in the project by default have data access permission for the current level and sub-level projects.

Example: A role created in the first-level project Apartment Water Leak Detection System can access all data under that project. However, a role created in the second-level project North Building A can only access data from North Building A and its sub-level projects.

  1. Switch to the Roles tab.
  2. Click the Roles button.
  3. Fill in the role name, description and check the role permissions.
  4. Click the Save button to complete role creation.

Role Editing​

  1. Switch to the Roles tab.
  2. Click the Operation button of the role record that needs editing, and select Edit from the dropdown menu.
  3. Modify the role name, description and check the role permissions.
  4. Click the Update button to complete role updating.

Role Deletion​

  1. Switch to the Roles tab.
  2. Click the Operation button of the role record that needs to be deleted, and select Delete from the dropdown menu.
  3. Click the Delete button in the role deletion confirmation pop-up to complete role deletion.
